登入 / 帳號

  1. 開啟後台網址 /admin/，輸入 Email 與密碼後點擊「登入」。
  2. 登入後 Token 存於瀏覽器 localStorage，有效期間內重新整理不需再次登入。
  3. 點擊側欄底部「登出」可清除 Token 並返回登入頁。
自動登入：後台登入後，直接開啟 /pos/、/quick-io/、/kitchen/ 會自動沿用同一憑證，無需重新輸入帳密。
預設帳號：首次啟動系統時自動建立帳號 admin / 密碼 admin，請登入後立即修改。

產品分類

後台側欄 → 產品分類。用於將產品歸類，可在產品查詢時篩選。

  1. 點擊「＋ 新增分類」，輸入分類名稱。
  2. 可修改名稱或刪除（刪除前需確認無產品使用此分類）。
  3. 分類也會一同匯出 / 匯入於產品 JSON 檔案中。

產品資料

後台側欄 → 產品資料。

新增 / 編輯產品

  1. 點擊「＋ 新增產品」，填入各欄位後儲存。
  2. 點擊列表中的「編輯」可修改現有產品。
欄位說明
SKU唯一識別碼，同款商品不同規格各有獨立 SKU，可點擊「自動產生」
條碼商品條碼，可用條碼機掃描輸入，也可點擊「自動產生」
名稱 / 分類 / 單位基本資料
成本價 / 售價用於庫存金額計算與報表
最低庫存低於此值時儀表板顯示「低庫存」警示；填 0 = 不限制
最高庫存高於此值時顯示「超量」警示；填 0 = 不限制
狀態停用後不顯示於 POS 菜單品項選擇

匯出產品（JSON）

  1. 點擊「⬇ 匯出」，下載包含所有分類與產品的 JSON 檔。
  2. 檔名格式：product_YYYYMMDD_HHMMSS.json。

匯入產品（JSON）

  1. 點擊「⬆ 匯入」，選擇 JSON 檔案（格式需與匯出相同）。
  2. 系統比對 SKU：已存在的產品更新資料、不存在的新增；分類已存在則略過。
  3. 匯入完成後庫存警示自動重整。
建議先匯出現有資料作為範本，再依格式批次編輯後匯入。

倉庫管理

後台側欄 → 倉庫管理。支援多倉庫，同一產品可在不同倉庫各自存放庫存。

  1. 點擊「＋ 新增倉庫」，輸入名稱與說明後儲存。
  2. 點擊倉庫列表中的倉庫名稱，展開該倉庫的「儲位管理」。
  3. 可在倉庫下新增儲位（Location），用於更細緻的庫存分區。

庫存查詢

後台側欄 → 庫存查詢。可依倉庫或產品篩選，顯示即時庫存量。

盤點調整

  1. 點擊「盤點調整」按鈕。
  2. 選擇產品與倉庫，輸入實際盤點數量（非異動量）。
  3. 系統自動計算差異並建立「盤點調整」移動紀錄。

入庫管理

後台側欄 → 入庫管理。建立正式採購 / 進貨單，完整追蹤進貨流程。

  1. 點擊「＋ 新增入庫單」，選擇目標倉庫，填入備註。
  2. 點擊「＋ 新增品項」，選擇產品（可用條碼掃描或搜尋），輸入數量與單價。
  3. 確認無誤後點擊「確認入庫」，庫存立即增加。
狀態說明
草稿可繼續編輯品項，尚未異動庫存
已入庫庫存已增加，不可再修改
已取消作廢，不影響庫存

出庫管理

後台側欄 → 出庫管理。建立正式銷售 / 出貨單，流程與入庫單相同。

狀態說明
草稿可繼續編輯品項，尚未異動庫存
已出庫庫存已減少，不可再修改
已取消作廢，不影響庫存

庫存移動紀錄

後台側欄 → 庫存移動。顯示所有產品相關的庫存異動歷史。

移動類型觸發來源
入庫確認入庫單、快速入庫
出庫確認出庫單、快速出庫
消耗快速消耗
盤點調整庫存查詢 → 盤點調整
POS 銷售、顧客點單觸發的庫存異動僅系統內部記錄，不顯示於此清單（避免與正式出庫單混淆）。

快速出入庫

網址：/quick-io/，或後台側欄 → 快速出入庫（開新分頁）。

  1. 選擇倉庫與操作類型（入庫 / 出庫 / 消耗）。
  2. 點擊「＋ 加入品項」，搜尋產品並輸入數量，可加入多筆。
  3. 點擊「確認送出」完成批次異動，系統自動建立移動紀錄。
適合日常快速補貨 / 領料，不需建立正式入出庫單的情境。

菜單管理

後台側欄 → 菜單管理。管理 POS 收銀台與顧客點單頁面顯示的菜單、品項與客製化選項。

建立菜單

  1. 點擊「＋ 新增菜單」，輸入菜單名稱後儲存。
  2. 點擊菜單名稱進入菜單詳細設定，包含「菜單分類」、「品項」、「選項群組」三個區塊。

菜單分類

在菜單內建立分類（如：飲料、主食、甜點），POS 點餐介面依分類顯示品項。

菜單品項

  1. 點擊「＋ 新增品項」，填入品項名稱、所屬分類、售價。
  2. 可上傳品項圖片（顯示於顧客點單頁面）。
  3. 勾選「連動庫存」後選擇對應產品，POS 或顧客出單時自動扣減產品庫存。
  4. 可為品項套用「選項群組」（如：冰塊、甜度）。

選項群組（客製化）

  1. 在菜單下建立選項群組（如：「甜度」），設定各選項（如：全糖、少糖、無糖）。
  2. 每個選項可附加額外費用。
  3. 品項套用選項群組後，POS 點餐或顧客點單時自動彈出客製化選項供顧客選擇。

預設菜單（進入頁面自動展開）

每次進入菜單管理頁面，可設定自動展開哪一張菜單，免去手動點選的步驟。

  1. 菜單列表每行最右側有一個 ☆ 星號按鈕。
  2. 點擊後變為 ★ 金色，菜單名稱旁出現「預設」橙色標籤，表示已設為預設。
  3. 再次點擊可取消預設。
  4. 設定儲存於瀏覽器（localStorage），僅影響當前瀏覽器，不影響其他使用者。
若預設菜單已被刪除，頁面會自動清空選擇，不會顯示錯誤。

品項批次客製化

  1. 勾選品項列表左側的核取方塊（可全選）。
  2. 點擊「批次客製化」按鈕，開啟批次設定視窗。
  3. 點擊每個選項組的狀態標籤循環切換： 不改（保持各品項現有設定）→ 套用（加入全部選中品項）→ 移除（從全部選中品項移除）。
  4. 點擊「儲存變更」，系統平行更新所有選中品項。

菜單匯入 / 匯出

  • 匯出所有菜單：點擊「匯出全部」，下載包含所有菜單的 JSON。
  • 匯入：點擊「匯入」，選擇 JSON 檔案覆蓋現有菜單資料。
  • 單一菜單匯出：進入菜單後點擊「匯出此菜單」。

收銀台操作

網址：/pos/，適合平板或行動裝置。後台登入後自動沿用登入憑證。

點餐結帳流程

  1. 左側選擇菜單分類，點擊品項加入購物車（右側）。
  2. 點擊購物車中的品項可調整數量或刪除。
  3. 點擊「結帳」按鈕，開啟結帳視窗。
  4. 選擇付款方式；現金付款可輸入收取金額，系統自動計算找零。
  5. 可點擊折扣快捷按鈕（後台設定）或手動輸入折扣。
  6. 點擊「確認付款」完成訂單，連動庫存品項自動扣庫。

折扣說明

折扣類型說明
折扣%例如 90 = 九折（小計 × 0.9）；填 0 = 免費贈送（全額折抵）
固定折抵輸入折抵金額，例如 50 = 折抵 $50

收現計算機鍵盤

現金付款時，結帳視窗提供計算機格式的收現輸入介面：

按鈕 / 區塊功能

+/− 切換面額按鈕模式：+ 為累加，− 為扣減（誤加可切換 − 後點擊面額扣回）
1000 / 500 / 100 / 50快速累加或扣減常見面額到收現金額（依目前 +/− 模式）

⌫刪除最後一位數字
C清除收現金額歸零

找零金額即時顯示於鍵盤下方（紅字＝金額不足，綠字＝可找零）。確認結帳時若金額不足系統會提示。

退款

  1. 後台側欄 → 銷售記錄，找到該筆訂單。
  2. 點擊「退款」，確認後庫存回補、訂單標記為已退款。
退款後庫存自動回補，請確認實際商品已收回再操作。

電子發票

顧客點單頁面

網址：/order/。顧客無需登入即可直接瀏覽菜單並自助點餐， 適合放置 QR Code 於桌面讓顧客掃碼使用。

顧客操作流程

  1. 顧客掃描 QR Code（或輸入網址），進入點單頁面。
  2. 畫面自動載入目前啟用的菜單，依分類顯示品項與圖片。
  3. 點擊品項，彈出客製化選項（甜度、冰塊等），確認後加入購物車。
  4. 購物車可調整數量、刪除品項，右下角顯示合計金額。
  5. 填入桌號（或外帶編號），點擊「送出訂單」。
  6. 送出後顯示訂單編號，顧客可憑此號碼取餐。
顧客點單頁面使用的菜單由系統設定 → 顧客點單菜單決定；若未設定則自動使用第一個啟用中的菜單。

訂單管理

後台側欄 → 訂單管理（需在頁面顯示管理中啟用）。查看與管理顧客點單訂單。

操作紀錄

每次狀態變更自動記錄操作者與時間，顯示於訂單列表的「操作紀錄」欄，方便追蹤各階段責任歸屬。

完成訂單自動建立銷售記錄

訂單狀態更新為「已完成」時，系統自動在銷售記錄中新增一筆對應記錄（來源標記為 cust_order），無需手動輸入，確保財務數據完整。

篩選與統計

  • 可依狀態（全部 / 待處理 / 處理中 / 已完成 / 已取消）篩選。
  • 可依日期篩選，統計列顯示當日各狀態筆數與總金額。

QR 碼管理

後台側欄 → QR 碼管理。為各桌號產生含時效 Token 的 QR Code，顧客掃描後進入已自動識別桌號的點單頁面。 Token 有時效性，到期後系統自動刷新，確保舊 QR 碼無法繼續使用。

Token 時效設定

  1. 在「Token 設定」卡片輸入有效時數（預設 24 小時，最小 1 小時）後儲存。
  2. 點擊「立即刷新所有 Token」強制更新全部桌號的 Token，舊 QR Code 即刻失效。
  3. 頁面顯示上次刷新時間與預計下次自動刷新時間，方便排程重新列印桌卡。
刷新後所有舊 QR Code 立即失效，需重新列印或下載新 QR Code 貼於桌面。

管理桌號

  1. 輸入桌號後點擊「＋ 新增」，系統自動產生唯一隨機 Token。
  2. 每個桌號可啟用 / 停用（停用後對應 QR Code 無法點餐，顯示錯誤）。
  3. 點擊「複製連結」取得含 Token 的完整 URL，或「下載 QR」取得圖片。
  4. 不再使用的桌號點擊「刪除」移除。

Token 機制說明

機制說明
Token URL 格式/order/?t=TOKEN，TOKEN 為隨機安全字串（24 字元）
自動刷新（懶觸發）顧客載入菜單時，若距上次刷新超過設定時效，系統自動更新全部 Token
手動刷新管理員可在 QR 碼管理頁點擊按鈕立即刷新並可同時修改時效
停用桌號停用後對應 Token 驗證失敗，顧客看到「此桌位目前停用」提示
過期提示Token 到期後顯示「QR 碼已過期，請洽服務人員重新掃描」
向下相容若尚未設定任何桌號 Token，舊式 /order/?table=桌號 仍可使用

廚房顯示

網址：/kitchen/。深色主題的備餐顯示介面， 適合放置於廚房顯示器。後台登入後直接開啟即可使用， 或輸入 kitchen_token（帳密登入）。

介面說明

  • 顯示所有「待處理」與「處理中」的訂單卡片，依下單時間由早到晚排列（FIFO）。
  • 每張卡片顯示：訂單編號、桌號、品項（依名稱字母排序）、等待時間、建立時間。
  • 等待時間格式：X天X時X分X秒，超過 10 分鐘顯示紅色警示。
  • 畫面頂部顯示即時時鐘，以及各狀態訂單數量統計。
  • 透過 SSE（Server-Sent Events）即時推送，每秒自動更新，無需手動重整。

員工操作

目前狀態可執行操作說明
待處理 開始處理 / 取消 點「開始處理」→ 變更為「處理中」
處理中 完成 / 取消 點「完成」→ 訂單從畫面移除（已完成）

外送訂單

後台側欄 → 外送訂單。管理來自外送平台的訂單。

  1. 訂單由外送平台透過 Webhook 自動推送，或點擊「同步」手動拉取。
  2. 點擊訂單查看品項明細。
  3. 更新訂單狀態（備餐中 → 配送中 → 已送達 / 已取消）。

外送平台設定

後台側欄 → 平台設定。設定各外送平台的 API 金鑰與 Webhook。

  1. 選擇平台，填入 API 金鑰後儲存。
  2. 複製「Webhook 接收網址」填入外送平台後台，即可接收即時訂單推送。
  3. 設定完成後可點擊「同步菜單至平台」，將 WMS 菜單同步到外送平台。

環境設定（.env）

系統部署時透過專案根目錄的 .env 檔案設定環境參數（由 .env.default 複製後修改）。

Port 設定

變數預設值說明
FLASK_PORT5000 Flask 內部 Port。若 5000 被佔用（macOS AirPlay），可改為 5001；nginx 會自動跟著連對應 Port
APP_PORT80 nginx 對外監聽 Port，使用者實際連線的 Port

SSL / 網域設定

變數預設值說明
NGINX_MODEhttp nginx 模式：
http — 純 HTTP，本機 / IP 直連使用
cloudflare — Cloudflare Origin CA SSL
https-letsencrypt — Let's Encrypt SSL
DOMAIN_ 對外域名。_ 代表接受任何域名（IP 直連）；啟用 SSL 時填入實際域名
CF_CERT_DIR./conf/nginx/certs/cloudflare NGINX_MODE=cloudflare 時，存放 origin.pem 與 origin.key 的目錄

JWT Token 設定

變數預設值說明
JWT_ACCESS_TOKEN_EXPIRES_HOURS8 Access Token 有效時數；超時需重新登入
JWT_REFRESH_TOKEN_EXPIRES_DAYS30 「記住我」用的 Refresh Token 有效天數

Cloudflare SSL 設定步驟

  1. 至 Cloudflare Dashboard → SSL/TLS → Origin Server → Create Certificate，下載 .pem 與 .key。
  2. 將憑證放至 CF_CERT_DIR 指定目錄（預設 conf/nginx/certs/cloudflare/），檔名需為 origin.pem 與 origin.key。
  3. 在 .env 設定 NGINX_MODE=cloudflare、DOMAIN=your.domain.com。
  4. Cloudflare SSL/TLS 模式改為「Full (Strict)」。
  5. 執行 docker compose up -d nginx 重啟 nginx 使設定生效。
